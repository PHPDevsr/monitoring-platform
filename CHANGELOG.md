# Changelog

All notable changes to this project are documented here. The format is based on
[Keep a Changelog](https://keepachangelog.com/en/1.1.0/), and this project adheres
to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

Release notes for each version are generated automatically by
[git-cliff](https://git-cliff.org) from
[Conventional Commits](https://www.conventionalcommits.org/) when a version tag
(`v*`) is pushed: the workflow regenerates this file, commits it to `master`,
and publishes the matching GitHub Release. Do not hand-edit per-version sections
below — commit conventional messages (`feat:`, `fix:`, etc.) and push a tag to
produce a new release section.
